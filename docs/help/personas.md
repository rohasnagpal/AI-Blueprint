# Personas

Personas define role-based AI behavior without coding. They can influence tone, reasoning style, output format, constraints, and the type of analysis the assistant performs.

## Where to Find Personas

Open **Personas** from the sidebar.

The Personas screen shows built-in and custom personas grouped by category. Use **Create Persona** to add a custom persona.

## Where Personas Are Used

Personas can be used in:

- Typed chat
- Live voice
- Email draft replies
- Council agents
- Some blueprint workflows

Select the persona before starting the chat, voice session, draft, or workflow.

## Creating a Persona

A persona should include:

- Name
- Category
- Description
- System instructions
- Constraints
- Tags
- Optional output format

Good persona instructions explain what the assistant should do, what it should avoid, and what output style is expected.

## Examples

Useful legal personas include:

- Contract Reviewer
- Litigation Associate
- Arbitration Prep Analyst
- Evidence Analyst
- Partner Reviewer
- Plain-English Explainer
- Client Email Drafter
- Legal Researcher

## Personas in Live Voice

Live voice uses the selected persona when the session starts. If the user changes persona during an active voice session, stop and restart voice for the change to take effect.

## Personas in Email

Email drafting can use a default email persona or a selected persona per message. Use email-specific personas to control tone, caution, and drafting style.

## Best Practices

- Keep persona instructions concrete.
- Avoid contradictory constraints.
- Use legal personas for legal workflows.
- Use plain-language personas for client-facing summaries.
- Test custom personas in a low-risk chat before using them on important work.
